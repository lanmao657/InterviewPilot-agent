# Agent Guidelines

1. 回答用户问题时使用中文回答，除非用户明确要求使用其他语言。
2. 修改代码前先阅读相关文件和现有实现，优先沿用项目已有结构、命名、组件和接口风格。
3. 变更保持小而聚焦，只处理用户当前请求相关的问题，不顺手重构无关代码。
4. 不要回滚或覆盖用户已有改动；发现工作区有无关改动时保持原样。
5. 涉及前端交互时，确保错误提示清晰、简短、对用户有帮助，避免只暴露控制台错误或技术堆栈。
6. 涉及后端接口、数据库模型或认证流程时，同步检查 schema、路由、迁移和测试是否一致。
7. 本地开发默认使用 `.env` 中的 PostgreSQL 配置，启动后端前应确保数据库迁移已执行到最新版本。
8. 对用户可见的表单校验应尽量在前端先提示，同时保留后端校验作为最终保障。
9. 完成改动后根据影响范围运行对应验证：后端优先运行 `pytest`，前端优先运行 `npm test` 和 `npm run build`。
10. 不要把密钥、令牌、私有服务地址或 `.env` 中的敏感信息写入提交说明、日志或前端代码。
11. 如果错误表象和根因不一致，优先解释根因。例如 CORS 报错可能是后端 500 导致的浏览器表象。
12. 给用户最终反馈时简明说明改了什么、验证了什么，以及是否还有需要用户手动处理的环境事项。
